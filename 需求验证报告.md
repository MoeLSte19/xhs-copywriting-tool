### 🔴 致命风险：2026 小红书最严新规

这是本次验证最关键的发现，PRD v1.0 中未充分考虑：

**2026年4月，小红书明确划分了"AI辅助创作"和"AI违规滥用"：**

| 行为 | 平台态度 |
|------|---------|
| AI辅助创作（用工具提高效率） | ✅ 支持 |
| AI违规滥用（批量养号、虚假信息、AI造假） | ❌ 严厉打击 |
| **AI批量生成营销内容** | **❌ 第7类违规：AI广告营销类** |

**处罚机制：偶尔违规→限流警告；多次违规→笔记限流下架、账号禁言；全程AI托管→直接封禁。**

**对我们的影响：**
- 如果用户用笔记侠生成文案 → 直接粘贴发布 → 被平台检测出AI痕迹 → 笔记限流
- **用户会认为"工具害我被限流"→ 不续费 → 差评传播 → 产品死亡**

**✅ 解决方案：笔记侠必须内置「降AI味」功能，这不是加分项，是生存条件。**

## 三、PRD v1.0 必须修正的内容

基于需求验证，PRD v1.0 存在以下关键遗漏，必须修正：

### 修正 1：🔴 新增核心功能——「AI味检测+降AI味」

**原 PRD 状态：** 未提及  
**验证结论：** 这是生存条件，不是加分项

| 功能 | 说明 | 优先级 |
|------|------|--------|
| AI味检测 | 生成文案后自动检测AI痕迹浓度，给出评分 | **P0（新增）** |
| 一键降AI味 | 自动改写，消除AI痕迹，口语化重写 | **P0（新增）** |
| 合规提示 | 提醒用户"建议加入个人真实体验后再发布" | **P1（新增）** |

**技术实现：** 
- AI味检测：用第二个Prompt让LLM从"平台审核员"视角评分
- 降AI味：用DeepSeek V4 Flash做口语化改写（成本仍为0.002元/次）
- 参考嘎嘎降AI的效果：深度重组句式+加入具体场景感

## 七、验证后的下一步建议

### 🔥 最关键的行动：验证"降AI味"效果

在写代码之前，先手动验证：

1. **用 DeepSeek API 生成 5 篇小红书笔记**
2. **用 AutoXHS 的 AI味检测工具 打分**
3. **写一个"降AI味"Prompt，改写这5篇**
4. **再次检测，看分数是否提升**
5. **如果提升明显 → 开干；如果不明显 → 先解决Prompt问题再写代码**

这个验证只需要 1-2 小时，但能避免开发出"文案质量不过关"的产品。
