# 小红书爆款文案生成器 — 「笔记侠」

## 一、产品定义

### 1.1 产品名称
**笔记侠** — 小红书爆款文案一键生成器

### 1.2 一句话定位
**输入一句话，3秒生成过检爆款笔记**（"过检"= 通过AI味检测）

### 1.3 产品形态
- **MVP 阶段**：Web 应用（H5），可直接在手机浏览器打开
- **V2 阶段**：微信小程序（需要企业主体认证，MVP 先绕过）
- **原因**：小程序审核需要企业资质 + 300元/年认证费 + 审核2-7天，MVP 用 H5 可以当天上线

### 1.4 为什么不做微信小程序 MVP
| 因素 | 小程序 | H5 Web |
|------|--------|--------|
| 上线速度 | 审核 2-7 天 | 当天上线 |
| 主体要求 | 企业/个体工商户 | 无 |
| 认证费 | 300 元/年 | 0 |
| 支付接入 | 需企业资质 | 个人收款码即可验证 |
| 分享传播 | 微信内分享好 | 链接分享 |
| 结论 | V2 再做 | **MVP 首选** |

## 三、用户画像

### 3.1 核心用户（按优先级排序）

**画像 A：成长期博主（1K-10K粉）— 最核心用户 🥇**
- 年龄：22-32岁
- 特征：有了一定粉丝基础，需要稳定更新频率保持增长
- 痛点：**用AI写文案被限流**，手动改40-60分钟太费时，需要两个工具来回切（生成+降AI味）
- 当前方案：Kimi生成 + 手动改 + 嘎嘎降AI处理，每篇15-20分钟
- 愿付费金额：9.9-29.9元（替代嘎嘎降AI 2.4元/篇+生成工具）
- 触达渠道：小红书搜"AI文案被限流"、抖音教程、运营社群

**画像 B：新手素人博主（<1K粉）— 量大养免费版 🥈**
- 年龄：18-28岁
- 特征：刚开始做小红书，对着空白文档2小时憋不出标题
- 痛点：不会写、不知道爆款长什么样、怕AI味被限流
- 愿付费金额：9.9元（先免费版养，3次/天够用后付费解锁）
- 触达渠道：小红书搜索"文案怎么写"、抖音教程

**画像 C：小微商家/店主 — V2阶段重点 🥉**
- 年龄：25-40岁
- 特征：有自己的店铺/产品，需要大量种草笔记但请文案80-120元/篇太贵
- 痛点：不会拆解竞品、不会写标题、不会引导转化
- 当前方案：代写团队80-120元/篇，或自己写但质量差
- 愿付费金额：29.9-49元/月（替代代写成本省95%）
- 触达渠道：小红书搜"种草文案"、商家社群、同行推荐

### 3.2 不做的人群

- 大品牌/企业市场部（需求复杂，竞品已覆盖）
- MCN 机构（需要批量管理，不属于 MVP 范围）
- 代写从业者（专业用户，要求高，不适合 MVP 阶段）

## 五、技术方案

### 5.1 架构

```
用户（手机/电脑浏览器）
    ↓
  H5 前端（Vue.js / React）
    ↓
  后端 API（Node.js / Python Flask）
    ↓
  DeepSeek V4 Flash API（文案生成）
    ↓
  返回生成结果
```

### 5.2 技术栈

| 组件 | 选型 | 理由 |
|------|------|------|
| **前端** | Vue 3 + Vite | 轻量、Claude Code 擅长生成 |
| **后端** | Node.js + Express | 简单，适合 API 转发 |
| **AI 模型** | DeepSeek V4 Flash | ¥2/百万输出token，极低成本 |
| **部署** | Vercel（前端）+ Railway/Render（后端）| 免费额度足够 MVP |
| **数据库** | 先不做，用 localStorage + JSON 文件 | MVP 不需要复杂存储 |
| **域名** | 买个 .com 或 .cn | ~50元/年 |

### 5.3 AI Prompt 工程（核心壁垒）

**这是整个产品的灵魂。** Prompt 质量决定文案质量，而文案质量决定用户是否付费。

```markdown
## 系统提示词（核心，需要反复打磨）

你是一个小红书爆款文案专家，擅长写出高互动率的小红书笔记。

你必须遵循以下规则：

1. 【风格】使用小红书真实博主的语气，避免AI味：
   - 口语化、像在跟闺蜜聊天
   - 适当使用"真的绝了""姐妹们""冲""按头安利"等高频词
   - 但不要过度堆砌，保持自然

2. 【结构】每篇笔记必须包含：
   - 标题：8-15字，带数字/感叹号/悬念
   - 正文：3-5个短段落，每段2-3句
   - emoji：每个段落1-2个，放在句末
   - 话题标签：3-5个，放在文末

3. 【禁忌】
   - 不使用"作为一名AI"等元描述
   - 不使用过于书面/官方的措辞
   - 不超过500字（小红书最佳阅读长度）
   - 不编造具体数据/价格（除非用户提供了）

4. 【输出格式】
   直接输出可复制的纯文本，不需要markdown格式标记。
```

### 5.4 单次生成成本测算

| 项目 | 数值 |
|------|------|
| 单次输入 token | ~800（含系统提示词 + 用户输入） |
| 单次输出 token | ~500（一篇笔记） |
| V4 Flash 输入成本 | ¥0.8/百万token × 0.0008M = ¥0.00064 |
| V4 Flash 输出成本 | ¥2/百万token × 0.0005M = ¥0.001 |
| **单次总成本** | **约 ¥0.002（2厘钱）** |
| 1000次生成总成本 | **约 ¥2** |

**结论：API 成本几乎可以忽略不计。** 即使用户每天生成 100 次，月成本不到 1 元。

### 5.5 总启动成本

| 项目 | 费用 |
|------|------|
| 域名 | ¥50/年 |
| DeepSeek API 充值 | ¥10（够用数万次） |
| Vercel 前端部署 | 免费 |
| Railway 后端部署 | 免费（Hobby 计划） |
| **总计** | **约 ¥60** |

### 6.2 收入测算

**保守估计（MVP 上线第1个月）：**
- 日均访问 100 人
- 30% 试用免费版（30人）
- 5% 转化付费（5人 × 9.9元 = ¥49.5/天）
- 月收入 ≈ ¥1,485

**乐观估计（第3个月，有内容引流后）：**
- 日均访问 500 人
- 8% 转化付费（40人 × 9.9元 = ¥396/天）
- 月收入 ≈ ¥11,880

**关键假设：** 需要通过小红书/抖音内容引流，否则自然流量极低。

### 6.3 收款方式（MVP 阶段）

| 方式 | 说明 | 优缺点 |
|------|------|--------|
| **个人微信收款码** | 手动付款后解锁 | ✅ 零成本 ❌ 不可扩展 |
| **爱发电/面包多** | 第三方创作者赞助平台 | ✅ 支持付费内容 ❌ 抽成10% |
| **微信支付商户** | 正规支付接入 | ✅ 体验好 ❌ 需企业资质 |

**MVP 策略：** 先用爱发电验证付费意愿，有人付费后再考虑正规支付。

## 九、可行性验证总结

### 9.1 可行性评分

| 维度 | 评分 | 说明 |
|------|------|------|
| **技术可行性** | ⭐⭐⭐⭐⭐ | Claude Code + DeepSeek API，零编程基础也能做 |
| **成本可行性** | ⭐⭐⭐⭐⭐ | 启动成本 ~¥60，API 成本几乎为零 |
| **市场需求** | ⭐⭐⭐⭐☆ | 3.5亿月活 + 7800万创作者，需求明确，但竞品多 |
| **变现可行性** | ⭐⭐⭐⭐☆ | 9.9元低价验证已跑通（小猫补光灯模式），需引流 |
| **差异化空间** | ⭐⭐⭐☆☆ | 竞品功能多，需靠"3秒出稿+爆款味正"差异化 |
| **综合评分** | ⭐⭐⭐⭐☆ | **可行，值得做，关键在 Prompt 质量和流量** |

### 9.2 关键风险

| 风险 | 影响 | 应对 |
|------|------|------|
| **小红书AI检测政策收紧** | 用户文案被限流→归咎工具→差评 | **内置降AI味+合规提示，闭环保护** |
| Prompt 质量不够好，文案AI味重 | 用户不付费 | 用真实爆款笔记训练Prompt，反复打磨 |
| 流量获取困难 | 没人用 | 先在小红书/抖音发内容引流 |
| 竞品跟进"生成+降AI味" | 差异化缩小 | 先发优势+快速迭代，6个月窗口期 |
| DeepSeek API 服务不稳定 | 生成失败 | 备用：通义千问 API（免费额度） |
| 微信小程序需要企业资质 | V2受阻 | 先做 H5，验证后再注册个体户 |
